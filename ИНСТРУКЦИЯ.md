# Safe Pilot — Инструкция по запуску

## Что сделано

Telegram-бот для расчёта центровки и загрузки (Weight & Balance) для частных самолётов.
Пилот сам вводит и подтверждает все данные по ВС (вес, центровка, станции, топливо,
пределы). Бот ничего не ищет в интернете, не читает документы, не использует ИИ — только
детерминированные расчёты по введённым данным.

Что внутри:

- Профили ВС с историей ревизий (при обновлении данные не перезаписываются, а сохраняется
  новая версия).
- Мастер добавления/обновления ВС, настройка станций (места, багаж, топливо) и CG-огибающей.
- Расчёт Ramp / Takeoff / Landing с выводом статуса (В ПРЕДЕЛАХ / НА ГРАНИЦЕ / ВНЕ ПРЕДЕЛОВ).
- Рекомендации по корректировке загрузки (топливо, багаж, перемещение груза), если расчёт
  вышел за пределы.
- 31 автотест, БД (SQLite/PostgreSQL) с миграциями через Alembic, Docker.

## Как запустить (через Docker)

Понадобится установленный Docker.

1. Перейдите в папку проекта:
   ```bash
   cd /Users/antonishchenko/Documents/my_code/safe-pilot
   ```
2. Скопируйте пример конфига и укажите токен бота:
   ```bash
   cp .env.example .env
   ```
   Откройте `.env` и впишите новый токен от BotFather в строку:
   ```
   BOT_TOKEN=ваш_новый_токен
   ```
3. Запустите:
   ```bash
   docker compose up --build
   ```
   При старте контейнер сам применит миграции БД и запустит бота (long polling — отдельный
   публичный адрес не нужен). Данные сохраняются в Docker-томе `safe_pilot_data`, они не
   пропадут при перезапуске контейнера.
4. Остановить: `Ctrl+C`, либо в фоне — `docker compose up -d --build`, затем
   `docker compose down` для остановки.

## Как пользоваться ботом в Telegram

1. Найдите бота по его username и нажмите **/start**.
2. **Добавить ВС** — мастер по шагам запросит: бортовой номер, модель, базовый вес пустого
   ВС, центровку или момент, максимальные веса, станции (места/багаж/топливо с ARM и
   плотностью топлива), точки CG-огибающей (минимум 2 — можно пропустить кнопкой «Skip»,
   если данных пока нет; тогда для этого ВС будет проверяться только вес, а центровка будет
   помечена как «NOT EVALUATED», пока вы её не добавите позже через «Обновить ВС»), опционально
   — исходный документ. В конце — обязательное подтверждение и сохранение профиля.
3. **Выбрать ВС** — если ВС несколько, выберите активное (иначе оно спросится перед каждым
   расчётом).
4. **Новый расчёт** — бот запросит загрузку по каждой станции (пассажиры/багаж), топливо в
   каждом баке (начальное, расход на руление, расход в полёте, минимальный запас), покажет
   сводку для подтверждения и выдаст результат по Ramp/Takeoff/Landing со статусом и, при
   необходимости, вариантами корректировки загрузки.
5. **Обновить ВС** — выберите ВС, затем по каждому полю можно либо ввести новое значение,
   либо нажать «↩️ Keep current», чтобы оставить как было — не нужно вводить всё заново.
   Создаётся новая ревизия, старые расчёты остаются привязаны к прежней.
6. **Мои ВС / Архивировать ВС / История расчётов / Помощь / Отмена** — остальные пункты
   меню самоочевидны.

Каждый результат сопровождается пометкой **UNVERIFIED** и напоминанием сверить данные с
актуальным POH/AFM и документацией ВС — бот не гарантирует лётную годность и не заменяет
проверку пилотом.
