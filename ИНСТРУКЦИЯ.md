# SafetyPilot — краткая инструкция

SafetyPilot — Telegram-бот для детерминированного расчёта Weight & Balance. Бот ничего не ищет в интернете и не использует ИИ для авиационной математики. Все параметры самолёта вводит и подтверждает пилот по актуальной документации конкретного борта.

## Обычный расчёт

После сохранения профиля выберите **✈️ Calculate**. Сначала бот предложит два варианта:

- **Взлёт — быстро**;
- **Взлёт + посадка — расширенно**.

Быстрый расчёт запросит только:

1. общий вес на передних местах, lb;
2. общий вес на задних местах, lb, если станция настроена;
3. общий вес багажа, lb, если станция настроена;
4. общий объём usable fuel, US gal.

Результат показывает вес, запас до ограничения, CG, допустимый диапазон CG и конкретный статус.

Если баки имеют разные ARM, а известен только общий объём топлива, бот показывает возможный диапазон CG. Он не придумывает распределение топлива. При необходимости появится статус **EXACT TANK SPLIT REQUIRED** — тогда используйте **Advanced / Landing** и введите фактическое количество в каждом баке.

## Рекомендации

При выходе за ограничения бот может предложить только пересчитанный и проверенный вариант:

- переложить уже имеющийся багаж между багажными отсеками;
- убрать часть багажа;
- добавить разрешённый и закреплённый груз в багажный отсек, только если введён его максимальный вес;
- уменьшить количество топлива.

Бот **не предлагает пересаживать пассажиров**. Рекомендация уменьшить топливо не проверяет маршрут, юридический резерв и performance — это остаётся ответственностью пилота.

## Добавление самолёта

Откройте **🛩 Aircraft → Add Aircraft**.

### Quick Setup

Введите:

- tail number/идентификатор и модель;
- Basic Empty Weight;
- Basic Empty CG или Basic Empty Moment;
- Maximum Takeoff Weight;
- станции передних/задних мест и багажа с ARM;
- каждый топливный бак: ARM, usable capacity и подтверждённую плотность;
- строки CG envelope: `weight, forward limit, aft limit`.

### Advanced Setup

Дополнительно позволяет сохранить Max Ramp, Max Landing, Max Zero Fuel, известный useful load для проверки, производителя и nickname.

## Редактирование самолёта

Откройте **🛩 Aircraft → Update Aircraft**. Сохранение создаёт новую ревизию; старые расчёты остаются привязаны к старой ревизии.

На экране станций можно:

- изменить название;
- изменить тип станции;
- изменить ARM;
- изменить/очистить лимит багажного или custom-отсека;
- изменить usable capacity и плотность топливного бака.

Если старый профиль содержит `Fuel Aux Tanks` как `CUSTOM`, выберите **Change type → Fuel Tank**, затем укажите usable gallons и density. Топливная станция должна вводиться в галлонах, а не в фунтах.

## Advanced / Landing

Этот режим запрашивает каждую станцию и каждый бак отдельно. Для оценки посадки нужно ввести плановый расход по **каждому** баку. Вопрос `Minimum required fuel...` удалён.

Для регулируемой custom-станции вводите вес и фактический ARM так:

```text
25 / 90.5
```

Если расход на посадку пропущен, бот всё равно показывает полноценный результат на взлёт и отдельно пишет, что посадочное состояние не оценивалось.

## Запуск через Docker

```bash
cp .env.example .env
# вставьте новый BOT_TOKEN в .env
docker compose up --build -d
docker compose logs -f bot
```

Остановка:

```bash
docker compose down
```

Не используйте `docker compose down -v`, если не хотите удалить volume с базой.

## Локальный запуск

```bash
python3.12 -m venv .venv
. .venv/bin/activate
pip install -e '.[dev]'
alembic upgrade head
python -m app.main
```

Тесты:

```bash
pytest -q
```
